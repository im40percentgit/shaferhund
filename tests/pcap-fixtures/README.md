# tests/pcap-fixtures

Deterministic pcap files used to exercise Suricata in `podman compose` CI runs.
Both files are generated by `generate_fixtures.py` using Python stdlib `struct`
(no scapy, no root, no live capture required).

---

## benign.pcap

**Contents:** A single TCP/HTTP exchange between two RFC 5737 documentation
addresses (192.0.2.1 → 192.0.2.2:80).  The client sends `GET /hello HTTP/1.1`
and the server responds `200 OK` with body `Hello, world!`.

**Source:** Hand-crafted by `generate_fixtures.py`; no real network traffic was
captured.

**Expected Suricata behaviour:** No alerts.  The traffic matches no ET Open
signatures.  `eve.json` will contain a `flow` event but no `alert` events.

**Size:** ~530 bytes (5 Ethernet frames).

---

## malicious.pcap

**Contents:** A single TCP/HTTP exchange identical in structure to benign.pcap,
except the server response body is the EICAR test string:

```
X5O!P%@AP[4\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H*
```

The EICAR test string is a harmless, universally-recognised security tool test
pattern defined at https://www.eicar.org/download-anti-malware-testfile/.
It contains no executable code and is safe to store in version control.

**Source:** Hand-crafted by `generate_fixtures.py`.

**Expected Suricata behaviour:** One or both of the following ET Open signatures
should fire when Suricata processes this pcap with `-r malicious.pcap`:

| SID     | Rule name |
|---------|-----------|
| 2000356 | ET POLICY EICAR test string in HTTP response |
| 2000357 | ET POLICY EICAR test string in HTTP request body |

`eve.json` will contain an `alert` event with `signature_id` matching one of
the above SIDs and `severity` 3 or lower.

**Size:** ~656 bytes (5 Ethernet frames).

---

## Regenerating

```bash
python3 tests/pcap-fixtures/generate_fixtures.py
python3 tests/pcap-fixtures/validate_fixtures.py
```

Both scripts require only Python 3.8+ stdlib — no pip installs needed.
